
# Visualizing the Runs for Agent K

This guide explains how to visualize the experiment logs generated by Agent K. The visualization helps understand the setup and execution of Agent K across different types of competitions (CV, NLP, Tabular).

## 📊 How to Use the Streamlit Dashboard

To launch the dashboard and visualize the logs, run the following command:

```shell
root_log_path=...  # path to the directory containing output.jsonl
streamlit run ./dashboard/logs_analysis/simple_app.py --server.fileWatcherType none --server.headless True -- --root_log_path $root_log_path
```

Replace `...` with the path to the log directory that contains `output.jsonl`. This log file is generated during the run of Agent K and contains information needed for visualization.

### 🔍 Example

```shell
root_log_path=./workspace/bioresponse/logs/setup/seed_42
streamlit run ./dashboard/logs_analysis/simple_app.py --server.fileWatcherType none --server.headless True -- --root_log_path $root_log_path
```

## 🧠 Log Types by Competition Type

Depending on the type of competition Agent K ran on, the log structure will vary:

- **CV / NLP Competitions**
  - Two types of logs are available:
    - **Setup Logs**: Captures the initial task setup.
    - **Main Pipeline Logs**: Includes logs for preprocessing, training, evaluation, and inference stages.
  - Each log can be visualized separately via the dashboard.

- **Tabular Competitions**
  - Only **Setup Logs** are generated.
  - No separate logs for model training or evaluation stages are available.

## 📁 Directory Structure Example

```
./workspace/<competition_name>/logs/
├── setup/
│   └── seed_*/output.jsonl        # Setup logs
└── pipeline/                      # Only present for CV/NLP
    └── seed_*/output.jsonl        # Main pipeline logs
```
